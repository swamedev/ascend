## 1. Purpose

Patterns describe what the data says. Insights say what it means. While a pattern says `trend_down detected for completion_rate`, an insight says *"Your completion rate has been declining over the last 5 missions — you may be attempting content above your current level."*

The Insight Engine (F1.3) is the third analytical stage of the Cognitive Pipeline. It transforms detected patterns into human-readable, actionable statements. Every insight is deterministic, traceable to its source patterns, and free of natural language generation AI.

This document defines:
- The Insight data model and type taxonomy
- The template system that produces human-readable text from pattern data
- The Insight Engine architecture and component model
- The severity computation model
- The determinism guarantee that ensures reproducibility

---

## 2. Insight Model

Every insight is a structured record with the following schema:

```python
@dataclass
class Insight:
    id: str                       # ULID
    insight_type: InsightType     # Enum from taxonomy (Section 3)
    title: str                    # Short headline (≤ 80 chars)
    description: str              # Full explanation (≤ 500 chars)
    severity: Severity            # info | warning | critical
    confidence: float             # 0.0–1.0, derived from source patterns
    source_pattern_ids: list[str] # ULIDs of patterns that triggered this insight
    generated_at: str             # ISO 8601 UTC
    metadata: dict[str, Any]      # Template version, builder context, etc.
```

### 2.1 Field Semantics

| Field | Constraints | Notes |
|-------|-------------|-------|
| `id` | ULID, globally unique | Generated by the engine at creation time |
| `insight_type` | Must exist in taxonomy | Determines which template is used |
| `title` | ≤ 80 chars, no markdown | Generated by template, not authored |
| `description` | ≤ 500 chars, plain text | Complete human-readable explanation |
| `severity` | One of `info`, `warning`, `critical` | Computed from pattern confidence and type |
| `confidence` | Float in [0.0, 1.0] | Lowest confidence among source patterns |
| `source_pattern_ids` | Non-empty list | Must link to existing patterns |
| `generated_at` | UTC ISO 8601 | Set by the engine at generation time |
| `metadata` | JSON-serializable dict | Reserved for auditability and debugging |

## 3. Insight Type Taxonomy

Each `InsightType` defines what triggers the insight and which template produces the text.

| InsightType | Category | Trigger | Template |
|---|---|---|---|
| `declining_performance` | Warning | pattern `trend_down` on `completion_rate` | "Your completion rate has declined..." |
| `accelerating_growth` | Positive | pattern `acceleration` on `xp_gained` | "Your XP growth is accelerating..." |
| `consistency_break` | Warning | pattern `consistency_score` drops below threshold | "Your performance consistency has dropped..." |
| `strength_identified` | Positive | pattern `consistency_score` high (≥ 0.8) | "You consistently perform well in..." |
| `stagnation_warning` | Warning | pattern `stagnation` detected | "You have plateaued in..." |
| `milestone_achieved` | Achievement | pattern `threshold_crossing` on `xp_total` | "Congratulations! You reached..." |
| `improvement_opportunity` | Suggestion | pattern `performance_gap` detected | "Consider focusing on..." |
| `engagement_drop` | Warning | pattern `frequency_burst` decreases | "Your engagement has decreased..." |
| `streak_milestone` | Achievement | pattern `trend_up` on `streak_length` | "You're on a streak!..." |

### 3.1 Category Semantics

| Category | Purpose | Delivery |
|----------|---------|----------|
| Warning | Alerts the builder to a negative trend or risk | Prioritised in UI, may trigger notification |
| Positive | Reinforces desirable behaviour | Shown in dashboard, lower priority than warnings |
| Achievement | Celebrates a milestone or streak | Badge or celebratory card, prominent UI |
| Suggestion | Offers actionable guidance | Recommendation card, medium priority |

## 4. Insight Templates

Every `InsightType` has a registered template function that produces the `title` and `description` from pattern data. Templates are **deterministic, parameterized strings**. No natural language generation AI is used.

### 4.1 Template Protocol

```python
class InsightTemplate(Protocol):
    def render(
        self,
        pattern_data: dict[str, Any],
        builder_context: dict[str, Any],
    ) -> tuple[str, str]:
        """Returns (title, description). Both are plain text strings."""
        ...
```

### 4.2 Template Examples

```python
class DecliningPerformanceTemplate:
    def render(self, pattern_data, builder_context):
        metric = pattern_data["metric"]            # e.g., "completion_rate"
        decline_pct = pattern_data["decline_pct"]  # e.g., 0.25
        window_missions = pattern_data["window"]   # e.g., 5
        title = f"Your {metric.replace('_', ' ')} is declining"
        description = (
            f"Your {metric.replace('_', ' ')} has declined by "
            f"{decline_pct:.0%} over the last {window_missions} missions. "
            f"You may be attempting content above your current level."
        )
        return title, description


class MilestoneAchievedTemplate:
    def render(self, pattern_data, builder_context):
        milestone = pattern_data["milestone_name"]
        value = pattern_data["milestone_value"]
        title = f"Milestone Unlocked: {milestone}"
        description = (
            f"Congratulations! You reached {milestone} "
            f"({value:.0f} XP total). Keep up the great work!"
        )
        return title, description


class ImprovementOpportunityTemplate:
    def render(self, pattern_data, builder_context):
        skill = pattern_data["weakest_skill"]
        gap = pattern_data["performance_gap"]
        title = f"Opportunity to improve in {skill}"
        description = (
            f"Consider focusing on {skill} — "
            f"there is a {gap:.0%} performance gap "
            f"compared to your other skills."
        )
        return title, description
```

### 4.3 Template Registry

Templates are registered at startup in a dictionary keyed by `InsightType`:

```python
TEMPLATE_REGISTRY: dict[InsightType, InsightTemplate] = {
    InsightType.declining_performance: DecliningPerformanceTemplate(),
    InsightType.accelerating_growth: AcceleratingGrowthTemplate(),
    InsightType.consistency_break: ConsistencyBreakTemplate(),
    InsightType.strength_identified: StrengthIdentifiedTemplate(),
    InsightType.stagnation_warning: StagnationWarningTemplate(),
    InsightType.milestone_achieved: MilestoneAchievedTemplate(),
    InsightType.improvement_opportunity: ImprovementOpportunityTemplate(),
    InsightType.engagement_drop: EngagementDropTemplate(),
    InsightType.streak_milestone: StreakMilestoneTemplate(),
}
```

Each template is instantiated once and reused across all builders. Templates hold no state.

## 5. Insight Engine Architecture

### 5.1 Component Model

```
Pattern[]
    │
    ▼
┌─────────────────────────────────────────┐
│  Insight Engine                          │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │  InsightGenerator               │    │  Maps pattern types → InsightTypes
│  │  - pattern → type dispatch      │    │
│  │  - confidence aggregation       │    │
│  │  - severity computation         │    │
│  └──────────┬──────────────────────┘    │
│             ▼                           │
│  ┌─────────────────────────────────┐    │
│  │  InsightTemplate (protocol)      │    │  Renders title + description
│  │  - TEMPLATE_REGISTRY dispatch    │    │
│  │  - pure function (no state)     │    │
│  └──────────┬──────────────────────┘    │
│             ▼                           │
│  ┌─────────────────────────────────┐    │
│  │  InsightStore (protocol)        │    │  Persists and queries insights
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
    │
    ▼
Insight[]
```

### 5.2 InsightEngine

The `InsightEngine` is the orchestrator. It receives a list of detected patterns and produces a list of insights:

```python
class InsightEngine:
    def __init__(
        self,
        generator: InsightGenerator,
        store: InsightStore,
    ):
        self._generator = generator
        self._store = store

    def process(self, patterns: list[Pattern]) -> list[Insight]:
        insights = self._generator.generate(patterns)
        self._store.save_many(insights)
        return insights
```

The engine is stateless — all state lives in the `InsightStore`. The same patterns always produce the same insights (see Section 7).

### 5.3 InsightGenerator

The `InsightGenerator` holds the mapping logic from patterns to insight types and delegates text rendering to templates:

```python
class InsightGenerator:
    def __init__(self, templates: dict[InsightType, InsightTemplate]):
        self._templates = templates

    def generate(self, patterns: list[Pattern]) -> list[Insight]:
        insights = []
        for pattern in patterns:
            insight_type = self._map_pattern_to_type(pattern)
            if insight_type is None:
                continue
            template = self._templates[insight_type]
            title, description = template.render(
                pattern_data=pattern.data,
                builder_context=pattern.builder_context,
            )
            insights.append(Insight(
                id=generate_ulid(),
                insight_type=insight_type,
                title=title,
                description=description,
                severity=self._compute_severity(pattern),
                confidence=self._aggregate_confidence(pattern),
                source_pattern_ids=[pattern.id],
                generated_at=now(),
                metadata={
                    "template_version": "1.0",
                    "pattern_type": pattern.type,
                },
            ))
        return insights

    def _map_pattern_to_type(self, pattern: Pattern) -> InsightType | None:
        MAPPING = {
            "trend_down.completion_rate": InsightType.declining_performance,
            "acceleration.xp_gained": InsightType.accelerating_growth,
            "consistency_drop": InsightType.consistency_break,
            "consistency_high": InsightType.strength_identified,
            "stagnation": InsightType.stagnation_warning,
            "threshold_crossing.xp_total": InsightType.milestone_achieved,
            "performance_gap": InsightType.improvement_opportunity,
            "frequency_decrease": InsightType.engagement_drop,
            "trend_up.streak_length": InsightType.streak_milestone,
        }
        return MAPPING.get(pattern.type)

    def _aggregate_confidence(self, pattern: Pattern) -> float:
        return pattern.confidence  # Single pattern → pass through

    def _compute_severity(self, pattern: Pattern) -> Severity:
        # See Section 6
        ...
```

### 5.4 InsightStore

```python
class InsightStore(Protocol):
    def save(self, insight: Insight) -> None: ...
    def save_many(self, insights: list[Insight]) -> None: ...
    def get_by_id(self, insight_id: str) -> Insight | None: ...
    def list_by_builder(
        self, builder_id: str,
        insight_type: InsightType | None = None,
        limit: int = 50, offset: int = 0,
    ) -> list[Insight]: ...
    def list_recent(
        self, builder_id: str,
        limit: int = 10,
    ) -> list[Insight]: ...
```

Storage follows the same SQLite conventions as ARCH-0040 (Signals) and ARCH-0035:

## 6. Severity Model

Severity is computed from the source pattern's type and its confidence. The mapping is deterministic:

| Pattern Category | Confidence Threshold | Severity |
|---|---|---|
| Warning (declining_performance, consistency_break, stagnation_warning, engagement_drop) | ≥ 0.8 | `critical` |
| Warning | 0.5 – 0.79 | `warning` |
| Warning | < 0.5 | `info` |
| Positive (accelerating_growth, strength_identified) | Any | `info` |
| Achievement (milestone_achieved, streak_milestone) | Any | `info` |
| Suggestion (improvement_opportunity) | Any | `info` |

### 6.1 Severity Computation

```python
def _compute_severity(self, pattern: Pattern) -> Severity:
    WARNING_TYPES = {
        InsightType.declining_performance,
        InsightType.consistency_break,
        InsightType.stagnation_warning,
        InsightType.engagement_drop,
    }
    insight_type = self._map_pattern_to_type(pattern)
    if insight_type in WARNING_TYPES:
        if pattern.confidence >= 0.8:
            return Severity.critical
        elif pattern.confidence >= 0.5:
            return Severity.warning
        else:
            return Severity.info
    return Severity.info
```

Only warning-category patterns can produce `warning` or `critical` severity. Positive, achievement, and suggestion insights are always `info`. This ensures that the builder is not overwhelmed with alarmist notifications for encouraging trends.

### 6.2 Severity Override

Metadata may contain an `override_severity` field for manual tuning. This is reserved for operator configuration and is never set by the engine itself. When present, the override takes precedence over the computed value.

## 7. Determinism Guarantee

Given the exact same sequence of patterns, the Insight Engine always produces the identical sequence of insights — same titles, same descriptions, same severity, same confidence.

This is guaranteed by:

- **Templates are pure functions**: same `pattern_data` and `builder_context` → same `(title, description)`.
- **Mapping is deterministic**: same pattern type → same `InsightType`.
- **Confidence aggregation is deterministic**: same pattern confidence → same insight confidence.
- **Severity computation is deterministic**: same pattern type + same confidence → same severity.
- **No AI**: zero NLG, zero LLM calls, zero probabilistic text generation.

### 7.1 Testing Determinism

```python
engine = InsightEngine(
    generator=InsightGenerator(templates=TEMPLATE_REGISTRY),
    store=InMemoryInsightStore(),
)

patterns = load_fixture("patterns_001.json")
insights_a = engine.process(patterns)
insights_b = engine.process(patterns)

assert insights_a == insights_b  # Identical output
```

For integration tests, a deterministic ULID factory is injected to ensure IDs are also reproducible:

```python
engine = InsightEngine(
    generator=InsightGenerator(templates=TEMPLATE_REGISTRY),
    store=InMemoryInsightStore(),
    ulid_factory=DeterministicUlidFactory(seed=42),
)
```

## 8. Failure Modes

| Failure | Behavior |
|---------|----------|
| Unknown pattern type | Skip pattern (return None from `_map_pattern_to_type`); log DEBUG |
| Missing template in registry | Log ERROR; skip insight generation for this type |
| Template render error (missing key) | Log WARNING; use fallback template with pattern type name |
| Store unavailable | Buffer insights in memory (max 500); retry on next cycle |
| Empty pattern list | Return empty list; no-op |
| Confidence outside [0.0, 1.0] | Clamp to valid range; log WARNING |
